# PRD 累计补充

## 0. 需求分册索引

- 登录业务需求文档：[docs/login-business-requirements.md](docs/login-business-requirements.md)

## 1. 产品形态与适用平台

- 前端产品是系统管理员管理后台。
- 支持以下形态：
  - PC / Mac / Linux 桌面系统中的浏览器网页。
  - 桌面应用客户端。
  - iPad 应用。
  - Android Pad 应用。
- 明确不做以下形态：
  - 手机 App。
  - 手机浏览器 H5。

## 2. 时间格式规范

- 所有“年月日”展示统一使用 `dd/mm/yyyy`。
- 所有“年月日 时分秒”展示统一使用 `dd/mm/yyyy hh:mm:ss`（24 小时制）。
- 示例：
  - 日期：`09/07/2026`
  - 日期时间：`09/07/2026 08:21:56`

## 3. 国家同步交互规则

- 点击国家同步按钮后，先展示确认弹窗。
- 用户确认后，进入“同步进行中”弹窗。
- 同步结束后，在同一弹窗体系内展示结果。
- 同步进行中，不允许关闭同步弹窗。
- 同步进行中，不允许刷新浏览器页面。
- 同步进行中，不允许关闭浏览器窗口。
- 面向未来桌面应用客户端，同步进行中同样不允许关闭软件客户端。

## 4. 当前实现说明

- 浏览器网页场景下，受浏览器安全机制限制，无法对“刷新页面/关闭浏览器”做到绝对强制拦截。
- 当前版本采用以下方式尽最大能力满足需求：
  - 同步进行中禁用同步弹窗右上角关闭按钮。
  - 拦截 `F5`、`Ctrl/Cmd + R` 刷新快捷键。
  - 在页面刷新或关闭前触发浏览器原生离开确认拦截。
- 后续若封装为桌面应用客户端，应在客户端外壳层（窗口关闭事件）继续落实“强制不允许关闭”的要求。
